    最近工作也是用个人的电脑，经常在提交时弄混工作和个人账号。全局做一下配置。

    参考：https://zhuanlan.zhihu.com/p/379982981

    全局配置 git 在不同目录下使用不同的 username #

    使用 includeIf 条件配置 #

    1. 创建不同环境的配置文件 在用户根目录下创建两个配置文件：
    1
2
3
4

    ## 工作配置
    touch ~/.gitconfig-work
## 个人配置
    touch ~/.gitconfig-personal

    1. 编辑配置文件

      • 工作配置 (~/.gitconfig-work)：
      1
2
3

          [user]
        name = Your Work Name
        email = work@example.com
      • 个人配置 (~/.gitconfig-personal)：
      1
2
3

          [user]
        name = Your Personal Name
        email = personal@example.com

    2. 修改全局配置 (~/.gitconfig) 添加条件包含规则：

      1
2
3
4

      [includeIf "gitdir:~/work/"]
    path = ~/.gitconfig-work
[includeIf "gitdir:~/personal/"]
    path = ~/.gitconfig-personal
      • gitdir:~/work/ 表示所有位于 ~/work/ 及其子目录的仓库会加载对应配置。
      • 使用 **/ 可匹配任意子目录（如 gitdir:~/work/**/）。

    验证 # 

    1
2
3
4
5
6

    # 查看当前生效的用户名和邮箱
git config --get user.name
git config --get user.email

# 查看所有配置来源（显示加载的配置文件路径）
git config --list --show-origin

    修改历史 commit #

    可能发现时已经用错误的账号提交了很多 commit 了，可以用如下方法修改：

    启动交互式 rebase #

    假设要修改最近的 3 次提交，可以运行：

    1

       git rebase -i HEAD~3

    如果需要修改第一个提交

    1

    git rebase -i --root

    选择要修改的提交 #

    在打开的编辑器中，会看到类似以下内容：

    1
2
3

       pick abcdef1 Commit message 1
   pick abcdef2 Commit message 2
   pick abcdef3 Commit message 3

    将要修改的提交前的 pick 改为 edit，例如：

    1
2
3

       edit abcdef1 Commit message 1
   pick abcdef2 Commit message 2
   pick abcdef3 Commit message 3

    保存并退出编辑器 #

    Git 会停在选择的提交上。

    更改用户名 #

    运行以下命令来更改提交的用户名和邮箱，并继续 rebase：

    1
2

       git commit --amend --author="New Name <new_email@example.com>"
   git rebase --continue

    如果有多个需要修改的提交，重复本步骤。
